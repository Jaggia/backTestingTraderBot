# Concept: OCC Option Symbol Format

Every US equity option is identified by an **OCC symbol** (also called an OSI symbol) — a standardised string used by all US exchanges, brokers, and data providers.

## Anatomy

```
QQQ   260228C00450000
│     │     │ │
│     │     │ └── Strike × 1000, zero-padded to 8 digits
│     │     └── Option type: C = Call, P = Put
│     └── Expiry: YYMMDD
└── Underlying root, left-padded to 6 characters with spaces
```

**Full example:** `QQQ   260228C00450000`
- Underlying: QQQ
- Expiry: Feb 28, 2026
- Type: Call
- Strike: $450.00

## Strike Encoding

Strike = `int(strike * 1000)`, zero-padded to 8 digits:

| Strike | Encoded | In symbol |
|---|---|---|
| $450.00 | 450000 | `00450000` |
| $452.50 | 452500 | `00452500` |
| $100.00 | 100000 | `00100000` |
| $1500.00 | 1500000 | `01500000` |

## Root Padding

The underlying root is left-padded with spaces to 6 characters:

| Ticker | Padded root |
|---|---|
| QQQ | `QQQ   ` |
| AAPL | `AAPL  ` |
| SPY | `SPY   ` |
| TSLA | `TSLA  ` |

## Codebase

Generated by `build_occ_symbol()` in `src/options/strike_selector.py`:

```python
root = underlying.ljust(6)          # "QQQ   "
exp  = expiry.strftime("%y%m%d")    # "260228"
strike_int = int(round(strike * 1000))
return f"{root}{exp}{option_type}{strike_int:08d}"
# → "QQQ   260228C00450000"
```

## Alpaca's Format

Alpaca uses the same OCC standard but **without space padding**:

```
QQQ260228C00450000     ← Alpaca API
QQQ   260228C00450000  ← OCC canonical
```

The live runner strips spaces before sending to Alpaca:

```python
def _strip_occ(occ_symbol: str) -> str:
    return occ_symbol.replace(" ", "")
```

Databento uses the canonical padded format (with spaces) for OPRA.PILLAR options data.
